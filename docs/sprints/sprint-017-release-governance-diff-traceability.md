# Sprint 017: Release Governance Diff Traceability

- **Sprint ID**: `017`
- **Date**: 2026-03-06
- **Status**: Completed
- **Owner**: 900 Labs

## Context

Sprint 016 added release-gate parity for governance checklist enforcement. A remaining follow-up was to make release governance diff selection and enforcement context auditable from workflow summaries/artifacts.

## Goals

1. Expose release governance diff-base decisions in a durable trace format.
2. Publish enforcement context for release triage without log scraping.
3. Reuse existing governance enforcement logic rather than duplicate rules.

## In Scope

1. Add report output support to governance checklist verification script.
2. Emit governance diff context artifact in release workflow.
3. Publish governance trace details into release workflow job summary.
4. Update release runbook and changelog.

## Out of Scope

1. Runtime product behavior changes.
2. Governance policy threshold changes.
3. External notification integrations.

## Deliverables

1. Updated `scripts/verify-governance-sprint-checklist.sh`:
   - added optional `REPORT_PATH` output support
   - writes base/head, changed files, governance files, sprint docs evaluated, checklist reference matches, and result reason
2. Updated `.github/workflows/release.yml`:
   - added governance report generation via `REPORT_PATH=.tmp/release-governance-diff-context.txt`
   - added job summary block with diff base, enforcement outcome, and context payload
   - uploads `release-governance-diff-context-<tag>` artifact
3. Updated `docs/RELEASE.md`:
   - documented release governance trace outputs (artifact + summary)
4. Updated `CHANGELOG.md` in `[Unreleased]`.

## Risks and Mitigations

1. **Risk**: trace output diverges from enforcement rules.
   - **Mitigation**: report is generated by the same enforcement script used in CI/release checks.
2. **Risk**: artifact missing on failed runs.
   - **Mitigation**: artifact and summary steps run with `if: always()` and warn when file is absent.

## Decisions

1. Keep trace payload as plain text for portability and direct readability.
2. Add trace summary at workflow level for quick triage without artifact download.

## Follow-Ups

1. Include direct artifact URL in workflow summary once stable artifact links are available in step context.
2. Consider emitting machine-readable JSON alongside text report for external governance analytics.
